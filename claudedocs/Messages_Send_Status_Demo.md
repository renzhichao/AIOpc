## UI细节设计

### 字体样式

```css
/* 时间戳 */
text-xs: 12px
opacity-70: 70% 透明度

/* 状态文本 */
text-xs: 12px (与时间戳同大小)
opacity-70: 70% 透明度 (与时间戳一致)
ml-1: 左边距 4px (与时间戳间距)
```

### 颜色方案

| 状态 | 文字颜色 | CSS类 | 用途 |
|------|---------|-------|------|
| 发送中 | 灰色 | `text-gray-400` | 表示进行中 |
| 已发送 | 半透明白/灰 | `opacity-70` | 表示已完成，不抢眼 |
| 发送失败 | 红色 | `text-red-400` | 表示错误，需注意 |

### 布局结构

```tsx
<div className="flex items-center justify-end gap-1">
  {/* 时间戳 */}
  {formatTimestamp(timestamp)}

  {/* 已发送状态 (仅成功时显示) */}
  {isUserMessage && message.sendStatus === 'sent' && (
    <span className="text-xs opacity-70 ml-1">
      已发送
    </span>
  )}

  {/* 其他状态 (发送中/失败) */}
  {renderMessageStatus(message)}
</div>
```

## 性能优化

### 1. 状态更新优化
```typescript
// 使用临时ID避免全量重新渲染
const tempId = `temp-${Date.now()}`;

// 精确更新特定消息
setMessages((prev) =>
  prev.map((msg) => {
    if (msg.type === 'user_message' && msg.message_id === tempId) {
      return { ...msg, sendStatus: 'sent' as const };
    }
    return msg;
  })
);
```

### 2. 条件渲染优化
```tsx
{/* 仅在需要时渲染状态文本 */}
{isUserMessage && message.sendStatus === 'sent' && (
  <span>已发送</span>
)}
```

### 3. CSS优化
- 使用 Tailwind 内置类，无需额外CSS文件
- 避免内联样式，提升渲染性能
- 使用 opacity 而非颜色变化，减少重绘

## 设计参考

### 灵感来源
1. **飞书**: 小字体状态显示，简洁优雅
2. **微信**: 状态与时间戳同行，不干扰
3. **Telegram**: 发送状态清晰可见

### 差异化设计
- **更简洁**: 不使用图标，仅文字提示
- **更轻量**: 无需额外的图标库
- **更快速**: 直接渲染文字，性能更优

## 用户反馈预期

### 预期改进
1. ✅ **减少视觉干扰**: 不再有弹出通知
2. ✅ **提升阅读流畅度**: 状态不抢夺注意力
3. ✅ **符合使用习惯**: 与主流聊天应用一致
