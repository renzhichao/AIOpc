## 1. 执行摘要

### 1.1 项目背景

AIOpc Platform 是一个 AI 基础设施平台，正在从本地部署模式转型为云 SaaS 模式。当前项目处于快速迭代阶段，但 DevOps 基础设施严重滞后，导致多次生产环境事故：

- **2026-03-17**: 数据丢失事件（PostgreSQL volume 重建导致 5 个实例丢失 3 个）
- **2026-03-17**: 配置回归导致 OAuth 登录失败
- **2026-03-18**: InstanceRegistry 缓存问题导致实例状态错误

### 1.2 业务目标

建立稳定、可靠的 DevOps 流水线，实现：

1. **零配置错误**: 通过自动化验证消除配置回归
2. **自动化部署**: 从手动 30 分钟降低到自动 5 分钟
3. **快速恢复**: MTTR（平均恢复时间）从 2+ 小时降低到 <15 分钟
4. **数据安全**: 建立完善的备份和恢复机制

### 1.3 关键问题概述

| 问题类别 | 当前状态 | 影响 | 优先级 |
|---------|---------|------|--------|
| 配置管理混乱 | 10+ 个 .env 文件，内容不一致 | 配置回归，服务中断 | 🔴 P0 |
| 完全缺乏 CI/CD | 无自动化测试、构建、部署 | 手动部署，易出错 | 🔴 P0 |
| 部署流程不规范 | 20+ 脚本分散，无人调用 | 数据丢失，服务中断 | 🟡 P1 |
| 监控告警缺失 | 无 APM、无日志聚合、无告警 | 问题发现滞后 | 🟡 P1 |
| 备份恢复缺失 | 无自动备份，无恢复测试 | 数据无法恢复 | 🟡 P1 |

   - 模板文件: `/platform/.env.production.example`
   - 本地覆盖: `/platform/.env.production.local`（不提交）

3. **配置隔离**:
   - 所有真实密钥移出代码库
   - 使用 GitHub Secrets 或环境变量注入
   - `.env.local` 文件加入 `.gitignore`

4. **配置验证**:
   - 创建 `scripts/verify-config.sh` 验证脚本
   - 检查所有 21 个必需环境变量
   - 检测占位符值（`cli_xxx`, `CHANGE_THIS`, `placeholder`）

**验收标准**:
- [ ] 配置文件数量 ≤ 3 个
- [ ] 配置一致性 100%（无冲突值）
- [ ] 敏感信息隔离 100%（无真实密钥）
- [ ] 配置验证自动化（pre-commit + CI）

#### 3.1.2 CI/CD 自动化需求

**验收标准**:
- [ ] CI 流水线成功率 ≥ 95%

- **影响**: 中等（影响应用性能）
- **概率**: 中（40%）

- **缓解措施**:
