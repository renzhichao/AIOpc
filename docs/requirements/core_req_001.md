## 1. 需求背景

### 1.1 现状问题

经过与最终用户的深入沟通，发现原方案（本地部署4台服务器）存在以下问题：

**部署门槛过高**：
- ❌ 需要准备4台闲置服务器
- ❌ 需要专业的Linux运维能力
- ❌ 部署周期长（2-4周）
- ❌ 维护成本高（需要专人运维）

**使用复杂度高**：
- ❌ 需要配置DeepSeek API密钥
- ❌ 需要手动配置Skills和Tools
- ❌ 需要配置飞书集成
- ❌ 普通用户无法独立完成

**资源浪费**：
- ❌ 4台服务器规模过大（50-200人企业）
- ❌ 资源利用率低（初期）
- ❌ 扩展不灵活

### 1.2 用户核心诉求

用户明确表达了以下三个核心诉求：

1. **快速云端部署**：希望通过云端Docker虚拟机方式快速部署OpenClaw服务
2. **简单扫码绑定**：通过扫描二维码即可将飞书账号与OpenClaw实例关联
3. **开箱即用**：实例预设LLM API key和Skills，认领后立即可用

### 1.3 目标用户画像

| 用户类型 | 比例 | 核心诉求 | 技术能力 |
|----------|------|----------|----------|
| **个人用户** | 40% | 个人AI助手，低门槛 | 低（不会运维） |
| **小团队** | 40% | 团队协作，快速上手 | 中（会基础操作） |
| **中小企业** | 20% | 试用验证，成本控制 | 中-高（有IT人员） |

## 2. 核心诉求详解

### 2.1 诉求一：云端Docker虚拟机快速部署

#### 2.1.1 需求描述

**用户期望**：
- 在云端快速创建一个OpenClaw服务实例
- 使用Docker容器化技术，隔离性好
- 部署时间在分钟级（不是小时或天）
- 无需关心底层服务器运维

**核心价值**：
- ✅ 降低部署门槛：无需本地服务器
- ✅ 提升部署速度：分钟级启动
- ✅ 减少运维负担：云端托管
- ✅ 按需扩容：弹性资源

#### 2.1.2 技术方案概述

```
┌─────────────────────────────────────────────┐
│              云服务平台                      │
│  ┌───────────────────────────────────────┐  │
│  │   Docker 容器编排平台                  │  │
│  │                                       │  │
│  │  ┌──────────┐  ┌──────────┐          │  │
│  │  │ Instance │  │ Instance │  ...    │  │
│  │  │    #1    │  │    #2    │          │  │
│  │  └──────────┘  └──────────┘          │  │
│  │       ↓             ↓                 │  │
│  │  OpenClaw    OpenClaw               │  │
│  └───────────────────────────────────────┘  │
│                   ↓                         │
│            负载均衡 + 网关                    │
└─────────────────────────────────────────────┘
```

**部署选项**：

| 云平台 | 推荐度 | 说明 | 预估成本 |
|--------|--------|------|----------|
| **阿里云ECS + Docker** | ⭐⭐⭐⭐⭐ | 国内首选，稳定性高 | ¥300-800/实例/月 |
| **腾讯云轻量服务器** | ⭐⭐⭐⭐ | 性价比高，简单易用 | ¥200-500/实例/月 |
| **AWS Lightsail** | ⭐⭐⭐ | 国际化，但访问慢 | ¥300-600/实例/月 |
| **自建Docker Swarm** | ⭐⭐ | 灵活但需运维 | 成本不定 |

**推荐方案**：**阿里云ECS + Docker**
- 单实例：2核4G，¥300/月（个人/小团队）
- 高配置：4核8G，¥600/月（中小企业）

#### 2.1.3 实例规格建议

| 规格 | 配置 | 适用场景 | 月成本 |
|------|------|----------|--------|
| **基础型** | 2核4G, 40GB SSD | 个人用户，试用 | ¥300-400 |
| **标准型** | 4核8G, 80GB SSD | 小团队（5-10人） | ¥600-800 |
| **增强型** | 8核16G, 160GB SSD | 中小企业（10-50人） | ¥1,200-1,500 |

### 2.2 诉求二：扫码认领"龙虾"

#### 2.2.1 需求描述

**用户期望**：
- 扫描一个二维码，即可完成飞书账号与OpenClaw实例的绑定
- 绑定过程简单、直观、无需技术背景
- 一个用户可以认领多个"龙虾"（多个实例）
- 一个"龙虾"只能被一个用户认领

**核心价值**：
- ✅ 极简onboarding：3步完成绑定
- ✅ 降低认知负担：无需理解技术细节
- ✅ 提升用户体验：类似微信扫码登录

#### 2.2.2 用户流程设计

```
┌──────────┐      ┌───────────┐      ┌──────────┐      ┌──────────┐
│ 访问服务  │  →   │ 扫描二维码  │  →   │ 飞书授权  │  →   │ 认领成功  │
│ Landing  │      │   QR码     │      │  OAuth   │      │  绑定完成  │
└──────────┘      └───────────┘      └──────────┘      └──────────┘
   Step 1            Step 2             Step 3            Step 4

时间：<2分钟
操作：扫码+点击确认
门槛：零技术背景
```

**详细流程**：

**Step 1：访问服务入口**
```
用户操作：访问部署服务的Web页面
页面显示：
┌────────────────────────────────────┐
│     🦞 OpenClaw 龙虾认领平台        │
│                                    │
│  扫描下方二维码，认领您的专属龙虾  │
│                                    │
│      [QR CODE 码]                  │
│      实例ID: opclaw-xxxx           │
│      状态: 待认领                  │
│                                    │
│  💡 如何使用？                     │
│  1. 扫描二维码                     │
│  2. 飞书授权登录                   │
│  3. 开始使用您的AI助手             │
└────────────────────────────────────┘
```

**Step 2：飞书授权**
```
用户操作：用飞书扫描二维码
系统行为：
1. 跳转到飞书OAuth授权页面
2. 用户确认授权
3. 飞书回调授权码
```

**Step 3：实例绑定**
```
系统操作：
1. 验证授权码有效性
2. 检查实例是否可认领（未被占用）
3. 创建用户-实例绑定关系
4. 生成访问Token

用户看到：
┌────────────────────────────────────┐
│      🎉 恭喜，认领成功！           │
│                                    │
│  您的专属龙虾：[可爱的龙虾图标]    │
│  实例ID: opclaw-xxxx               │
│  认领时间: 2026-03-12 14:30        │
│                                    │
│  [立即使用] [查看文档]             │
└────────────────────────────────────┘
```

**Step 4：开始使用**
```
用户操作：点击"立即使用"
跳转到：飞书对话界面（已创建机器人对话）
或跳转到：Web管理界面
```

#### 2.2.3 二维码生成逻辑

```javascript
// 二维码内容格式
qr_content = {
  "action": "claim_instance",
  "instance_id": "opclaw-xxxxxxxx",
  "timestamp": 1678123456789,
  "signature": "xxxxx"  // 防篡改签名
}

// 二维码URL
qr_url = `https://openclaw.service.com/claim?instance_id=${instance_id}&token=${token}`

// 飞书OAuth URL
feishu_oauth_url = `https://open.feishu.cn/open-apis/authen/v1/authorize?` +
  `app_id=${app_id}&` +
  `redirect_uri=${redirect_url}&` +
  `scope=&` +
  `state=${instance_id}`
```

#### 2.2.4 实例状态管理

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| **待认领** | 实例已创建，等待用户认领 | 扫码认领 |
| **已认领** | 用户已绑定实例 | 正常使用 |
| **已停用** | 用户主动停用或欠费 | 重新激活 |
| **已释放** | 实例到期或用户主动释放 | 无法恢复 |

## 4. 功能需求

### 4.1 核心功能（Must Have）

#### 4.1.1 实例管理

**功能编号**：F-001
**功能名称**：实例生命周期管理

**功能描述**：
- 实例创建：自动分配Docker容器
- 实例查询：查看实例状态
- 实例启动：启动已停止的实例
- 实例停止：主动停止实例（节省费用）
- 实例释放：删除实例和数据
- 实例续费：延长实例使用期限

**API设计**：
```yaml
# 创建实例
POST /api/instances
Response: { instance_id, status, qr_code_url }

# 查询实例
GET /api/instances/{instance_id}
Response: { instance_id, status, created_at, expires_at }

# 停止实例
POST /api/instances/{instance_id}/stop

# 启动实例
POST /api/instances/{instance_id}/start

# 释放实例
DELETE /api/instances/{instance_id}
```

#### 4.1.2 二维码生成与验证

**功能编号**：F-002
**功能名称**：认领二维码管理

**功能描述**：
- 生成认领二维码：每个实例生成唯一QR码
- 二维码有效期：24小时（可配置）
- 二维码防篡改：数字签名验证
- 扫码统计：记录扫码次数和时间

**API设计**：
```yaml
# 获取实例二维码
GET /api/instances/{instance_id}/qr-code
Response: { qr_code_url, expires_at, signature }

# 验证二维码
POST /api/qr-codes/validate
Body: { qr_code_token, signature }
Response: { valid: true/false, instance_id }
```

#### 4.1.3 飞书OAuth集成

**功能编号**：F-003
**功能名称**：飞书授权登录

**功能描述**：
- OAuth 2.0授权流程
- 获取用户信息（用户ID、姓名、部门）
- 创建用户-实例绑定关系
- 生成访问Token

**飞书开放平台配置**：
```yaml
feishu:
  app_id: cli_xxxxxxxxx
  app_secret: xxxxxxxxxxxxxxxx
  redirect_uri: https://openclaw.service.com/oauth/callback
  scopes:
    - contact:user.base:readonly  # 读取用户基本信息
    - contact:user.email:readonly # 读取用户邮箱
```

**OAuth流程**：
```
1. 用户扫码
   ↓
2. 跳转飞书授权页
   URL: https://open.feishu.cn/open-apis/authen/v1/authorize
   参数: app_id, redirect_uri, state(instance_id)
   ↓
3. 用户同意授权
   ↓
4. 飞书回调redirect_uri
   参数: code, state
   ↓
5. 后端换取user_access_token
   API: https://open.feishu.cn/open-apis/authen/v3/oidc/access_token
   ↓
6. 获取用户信息
   API: https://open.feishu.cn/open-apis/authen/v3/user_info
   ↓
7. 创建绑定关系
   INSERT INTO user_instances (user_id, instance_id, bound_at)
   ↓
8. 返回认领成功
```
