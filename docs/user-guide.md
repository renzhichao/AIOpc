# 用户指南 - 远程实例认领和对话

## 概述

欢迎使用 OpenClaw 智能体平台！本指南将帮助您了解如何使用远程实例认领和对话功能。

**主要功能**：
- 查看可用的远程实例
- 认领未分配的实例
- 与认领的实例进行实时对话
- 管理您的实例集合

---

## 1. 如何查看可用实例

### 1.1 登录系统

使用飞书账号登录平台后，您将进入 Dashboard（仪表板）页面。

### 1.2 查看 Dashboard 统计

在 Dashboard 页面，您可以看到以下统计信息：

**![Dashboard 页面](../screenshots/dashboard.png)**

*图 1: Dashboard 页面显示实例统计*

**统计卡片说明**：
- **我的实例数**：您已认领的实例总数
- **可用实例数**：当前可认领的远程实例数量（如有新实例会显示黄色高亮）
- **运行中实例数**：当前正在运行的实例数量
- **健康实例数**：状态健康的实例数量

### 1.3 可用实例通知

当有可用实例时，Dashboard 会显示黄色通知横幅：

**![可用实例通知](../screenshots/unclaimed-notification.png)**

*图 2: 可用实例通知横幅*

通知内容包括：
- 可用实例数量
- 点击可跳转到实例列表

### 1.4 进入实例列表

有两种方式进入实例列表页面：

**方式一：从 Dashboard**
- 点击"查看我的实例"卡片
- 或点击"认领新实例"卡片
- 或点击可用实例通知横幅

**方式二：通过导航**
- 使用侧边栏导航进入"实例管理"

---

## 2. 如何认领实例

### 2.1 进入实例列表页面

实例列表页面有两个标签页：
- **我的实例**：显示您已认领的实例
- **可用实例**：显示可认领的远程实例

**![实例列表页面](../screenshots/instance-list.png)**

*图 3: 实例列表页面*

### 2.2 切换到"可用实例"标签页

点击页面顶部的"可用实例"标签按钮。

### 2.3 浏览可用的远程实例

可用实例列表显示以下信息：

**![可用实例卡片](../screenshots/unclaimed-instance-card.png)**

*图 4: 可用实例卡片*

**实例信息包括**：
- **实例 ID**：实例的唯一标识符
- **远程地址**：实例的主机地址和端口
- **版本信息**：实例运行的 OpenClaw 版本
- **健康状态**：
  - 🟢 健康：实例运行正常
  - 🟡 警告：实例存在潜在问题
  - 🔴 不健康：实例运行异常
- **能力标签**：实例支持的功能（如：聊天、文件操作等）

### 2.4 认领实例

找到您想要的实例后，点击实例卡片上的"认领"按钮。

**![认领按钮](../screenshots/claim-button.png)**

*图 5: 认领按钮位置*

**认领过程**：
1. 点击"认领"按钮
2. 按钮显示"认领中..."并禁用
3. 等待认领完成（通常 1-2 秒）
4. 认领成功后，实例会从"可用实例"列表移除
5. 实例自动出现在"我的实例"列表中

### 2.5 认领失败处理

如果认领失败，您会看到错误提示。可能的原因：
- 实例已被其他用户认领
- 网络连接问题
- 实例状态异常

**解决方案**：
- 刷新列表查看最新状态
- 检查网络连接
- 联系技术支持

### 2.6 刷新列表

点击右上角的"刷新"按钮可以更新实例列表。系统每 10 秒自动刷新一次。

## 3. 如何开始对话

### 3.1 在"我的实例"中找到实例

切换到"我的实例"标签页，找到您想对话的实例。

### 3.2 点击"开始对话"按钮

在实例卡片上，点击"开始对话"按钮。

**![开始对话按钮](../screenshots/start-chat-button.png)**

*图 6: 实例卡片上的开始对话按钮*

或者，您可以：
- 点击实例卡片进入实例详情页
- 在详情页点击"开始对话"按钮

### 3.3 进入对话页面

系统会自动跳转到对话页面，并建立 WebSocket 连接。

**![对话页面](../screenshots/chat-page.png)**

*图 7: 对话页面布局*

### 3.4 对话页面说明

**页面结构**：

**顶部栏**：
- **返回按钮**：返回实例列表
- **实例信息**：实例名称和类型标签
- **状态标签**：实例当前状态（运行中/已停止）
- **连接状态指示器**：
  - 🟢 已连接：WebSocket 连接正常
  - 🟡 连接中：正在建立连接
  - 🔴 未连接：连接断开

**消息区域**：
- 显示对话历史
- 用户消息显示在右侧（蓝色气泡）
- AI 响应显示在左侧（白色气泡）
- 每条消息显示时间戳
- 发送失败的消息会显示"(发送失败)"

**输入区域**：
- 文本输入框：支持多行输入
- 发送按钮：发送消息
- 快捷键提示：按 Enter 发送，Shift + Enter 换行

### 3.5 发送消息

**步骤**：
1. 在输入框中输入您的消息
2. 按 Enter 键或点击"发送"按钮
3. 消息立即显示在聊天区域（状态：发送中）
4. 等待 AI 响应（会显示"AI 正在输入..."指示器）
5. 收到响应后显示在聊天区域

**![发送消息](../screenshots/send-message.png)**

*图 8: 发送消息流程*

### 3.6 连接状态说明

**连接状态指示器**：
- **已连接**：可以正常发送和接收消息
- **连接中**：正在建立连接，请稍候
- **未连接**：无法发送消息，请检查网络或刷新页面

**自动重连**：
- 如果连接断开，系统会自动尝试重连
- 重连期间发送按钮会被禁用
- 重连成功后可以继续对话

### 3.7 多行输入

如果您需要输入多行文本：
- 按 Shift + Enter 换行
- 输入框会自动扩展高度（最多 200px）

## 4. 常见问题解答 (FAQ)

### 4.1 什么是远程实例？

**答**：远程实例是部署在远程服务器上的 OpenClaw 智能体实例。与本地实例不同，远程实例：
- 由平台管理员预先部署
- 不需要用户自己创建
- 可以被任何用户认领
- 认领后归属于特定用户

**主要区别**：
| 特性 | 本地实例 | 远程实例 |
|------|---------|---------|
| 创建方式 | 用户自己创建 | 管理员部署 |
| 资源消耗 | 消耗平台配额 | 独立资源 |
| 访问方式 | 通过平台代理 | 直接连接 |
| 成本 | 按使用计费 | 通常免费 |

### 4.2 为什么有些实例显示"可用"？

**答**："可用"状态表示该实例：
- 已经部署并运行
- 尚未被任何用户认领
- 可以被您认领使用

这些实例通常是：
- 测试环境
- 演示环境
- 共享资源
- 预配置的专用环境

### 4.3 认领实例后会发生什么？

**答**：认领实例后：
1. **实例归属**：实例归属到您的账户下
2. **独占使用**：其他用户无法再认领该实例
3. **永久使用**：除非您主动释放，否则实例一直属于您
4. **完全控制**：您可以启动、停止、重启实例
5. **对话权限**：您可以与实例进行实时对话

**注意**：认领后无法"取消认领"，如需放弃使用权，请联系管理员。

### 4.4 对话时连接断开怎么办？

**答**：如果连接断开：
1. **自动重连**：系统会自动尝试重连（最多 3 次）
2. **查看状态**：留意页面右上角的连接状态指示器
3. **手动刷新**：如果自动重连失败，刷新页面
4. **检查网络**：确保您的网络连接正常
5. **检查实例**：确认实例状态为"运行中"

**重连期间**：
- 发送按钮会被禁用
- 已发送的消息可能会丢失
- 重连成功后可以继续对话

### 4.5 可以同时与多个实例对话吗？

**答**：当前版本的限制：
- **同一时间**：只能与一个实例进行对话
- **多标签页**：可以在不同浏览器标签页打开不同实例的对话页面
- **未来计划**：我们正在开发支持同时多个对话的功能

**建议**：
- 使用浏览器标签页在不同实例间切换
- 每个标签页保持一个对话会话

### 4.6 如何释放已认领的实例？

**答**：当前版本不支持主动释放实例。如果您需要：
- **停止使用**：只需停止与该实例的对话
- **放弃实例**：联系平台管理员协助处理
- **未来功能**：我们计划在后续版本中添加释放功能

**注意**：
- 停止对话不会释放实例
- 实例仍归属于您的账户
- 其他用户无法访问您的实例

### 4.7 消息发送失败怎么办？

**答**：如果消息发送失败，您会看到"(发送失败)"标记。可能的原因和解决方案：

**可能原因**：
1. **网络问题**：检查您的网络连接
2. **实例状态**：确认实例正在运行
3. **连接断开**：查看连接状态指示器
4. **实例过载**：实例可能正在处理其他请求

**解决方案**：
1. **检查连接**：确认右上角显示"已连接"
2. **刷新页面**：重新建立连接
3. **重试发送**：复制消息内容后重新发送
4. **检查实例**：返回实例列表查看实例状态
5. **联系支持**：如果问题持续，联系技术支持

### 4.8 对话历史会保存吗？

**答**：
- **当前版本**：对话历史仅在当前会话中保存
- **刷新页面**：会清空对话历史
- **切换实例**：每个实例有独立的对话会话
- **未来计划**：我们正在开发对话历史持久化功能

**建议**：
- 重要信息请及时记录
- 不要依赖浏览器缓存保存对话

### 4.9 可以发送文件或图片吗？

**答**：
- **当前版本**：仅支持文本消息
- **未来计划**：我们计划支持文件上传、图片发送等功能
- **替代方案**：可以通过其他方式分享文件链接

### 4.10 实例响应慢怎么办？

**答**：如果实例响应较慢：
1. **网络延迟**：检查您的网络连接速度
2. **实例负载**：实例可能正在处理复杂任务
3. **模型推理**：AI 响应时间取决于请求复杂度
4. **等待时间**：通常响应在几秒到几十秒之间

**建议**：
- 耐心等待响应
- 避免重复发送相同消息
- 如果超时（>60秒），刷新页面重试

## 5. 故障排除

### 5.1 页面加载失败

**症状**：页面无法加载或显示错误信息

**解决方案**：
1. 刷新浏览器页面（F5 或 Cmd+R）
2. 清除浏览器缓存
3. 检查网络连接
4. 尝试使用无痕/隐私浏览模式
5. 联系技术支持

### 5.2 无法看到可用实例

**症状**："可用实例"标签页显示"暂无可用实例"

**可能原因**：
- 当前确实没有可用的远程实例
- 网络连接问题
- 权限不足

**解决方案**：
1. 点击"刷新"按钮更新列表
2. 检查网络连接
3. 联系管理员确认是否有可用实例
4. 查看 Dashboard 上的"可用实例数"统计

### 5.3 认领按钮无响应

**症状**：点击"认领"按钮后没有反应

**解决方案**：
1. 等待几秒，认领可能需要一些时间
2. 刷新页面查看实例是否已认领
3. 检查浏览器控制台是否有错误信息
4. 尝试认领其他实例
5. 联系技术支持

### 5.4 对话页面无法建立连接

**症状**：连接状态一直显示"连接中"或"未连接"

**解决方案**：
1. 刷新页面重新建立连接
2. 检查实例状态是否为"运行中"
3. 检查网络连接
4. 尝试使用其他浏览器
5. 联系技术支持

### 5.5 消息发送后没有响应

**症状**：消息显示已发送，但 AI 一直不回复

**可能原因**：
- 实例正在处理复杂任务
- 实例出现异常
- 网络连接问题

**解决方案**：
1. 耐心等待（最长 60 秒）
2. 查看连接状态是否正常
3. 尝试发送简短消息测试
4. 刷新页面重新连接
5. 联系技术支持

### 5.6 浏览器兼容性问题

**支持的浏览器**：
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

**不支持的浏览器**：
- Internet Explorer（任何版本）
- 旧版浏览器

**解决方案**：
- 更新浏览器到最新版本
- 使用推荐的浏览器
