## 2. 阿里云VPC设计

### 2.1 VPC网段规划

| 网络类型 | CIDR块 | 用途 | 说明 |
|----------|--------|------|------|
| VPC主网段 | 172.16.0.0/16 | 整体云上网络 | 支持65534个主机 |
| 公网子网 | 172.16.1.0/24 | 需要公网访问的资源 | SLB、NAT等 |
| 私网子网1 | 172.16.10.0/24 | ECS应用集群 | OpenClaw服务 |
| 私网子网2 | 172.16.20.0/24 | 数据库服务 | RDS、Redis等 |

### 2.2 子网详细配置

#### 公网子网 (172.16.1.0/24)

**用途**：承载需要公网访问的服务

**资源**：
- SLB (负载均衡)
- NAT网关
- VPN网关
- ECS (Bastion Host - 可选)

**安全组规则**：
```yaml
入站规则:
  - 端口: 443
    协议: TCP
    来源: 0.0.0.0/0
    用途: HTTPS访问

  - 端口: 80
    协议: TCP
    来源: 0.0.0.0/0
    用途: HTTP访问

出站规则:
  - 全部放行
```

#### 私网子网1 (172.16.10.0/24) - 应用层

**用途**：OpenClaw Agent服务集群

**资源**：
- ECS-1: 172.16.10.10 (Master)
- ECS-2: 172.16.10.11 (Slave)
- ECS-3: 172.16.10.12 (Slave)

**安全组规则**：
```yaml
入站规则:
  - 端口: 3000-3100
    协议: TCP
    来源: 172.16.1.0/24 (SLB)
    用途: Agent API访问

  - 端口: 22
    协议: TCP
    来源: 172.16.1.0/24 (Bastion)
    用途: SSH管理

出站规则:
  - 允许访问Internet (用于LLM API调用)
  - 允许访问私网子网2 (数据库)
  - 允许访问VPN网段 (本地机房)
```

#### 私网子网2 (172.16.20.0/24) - 数据层

**用途**：数据库和缓存服务

**资源**：
- RDS PostgreSQL: 172.16.20.10
- Redis: 172.16.20.20

**安全组规则**：
```yaml
入站规则:
  - 端口: 5432
    协议: TCP
    来源: 172.16.10.0/24 (应用层)
    用途: PostgreSQL访问

  - 端口: 6379
    协议: TCP
    来源: 172.16.10.0/24 (应用层)
    用途: Redis访问

出站规则:
  - 无出站规则 (数据库层)
```

## 3. VPN连接设计

### 3.1 IPsec VPN配置

#### 阿里云侧配置

**VPN网关**：
- 公网IP：自动分配
- VPN类型：IPsec VPN
- 本端网段：172.16.0.0/16

**用户网关**：
- 公网IP：本地机房出口IP
- 路由方式：静态路由

**IPsec连接**：
- 预共享密钥：StrongPSKKey2024!
- 加密算法：AES256
- 认证算法：SHA256
- DH分组：group14
- IKE版本：IKEv2
- SA生存周期：86400秒

#### 本地机房侧配置

**VPN设备/软件**：
- 公网IP：固定IP或动态DNS
- 内部网段：192.168.1.0/24
- 路由配置：
  ```
  到172.16.0.0/16 -> VPN隧道
  默认路由 -> 本地网关
  ```

### 3.2 路由规划

#### 静态路由表

**阿里云侧**：
```
目标网段          下一跳           类型
172.16.0.0/16    本地             直连
192.168.1.0/24   VPN网关          静态路由
0.0.0.0/0        NAT网关          默认路由
```

**本地机房侧**：
```
目标网段          下一跳           类型
192.168.1.0/24   本地             直连
172.16.0.0/16    VPN隧道          静态路由
0.0.0.0/0        本地网关         默认路由
```

## 4. 飞书集成网络架构

### 4.1 Webhook接收

```
飞书服务器
    │
    │ HTTPS Webhook
    ▼
阿里云 SLB (公网IP:443)
    │
    ▼
ECS应用集群 (私网IP:3000)
    │
    ▼
OpenClaw Agent服务
```

**网络流程**：
1. 用户在飞书发送消息
2. 飞书服务器通过Webhook推送到阿里云SLB
3. SLB分发到ECS集群中的某个实例
4. OpenClaw处理请求并调用DeepSeek API
5. 处理结果通过飞书API返回给用户

### 4.2 飞书API调用

**出站规则**：
```
ECS集群 -> 飞书API (https://open.feishu.cn)
```

**NAT配置**：
- 配置NAT网关实现ECS访问外网
- SNAT池：使用EIP或共享带宽包
- 带宽预估：10-20Mbps

## 5. 邮件集成网络架构

### 5.1 SMTP发送

```
ECS应用集群
    │
    │ SMTP (TCP 25/465/587)
    ▼
NAT网关
    │
    ▼
邮件服务器 (企业邮箱/SMTP服务)
```

**配置要点**：
- SMTP端口：25（标准）、465（SSL）、587（TLS）
- 使用企业邮箱或SMTP服务（如阿里云邮件推送）
- 认证方式：用户名+密码或OAuth

### 5.2 IMAP接收（可选）

如果需要邮件监听功能：

```
邮件服务器
    │
    │ IMAP (TCP 143/993)
    ▼
VPN隧道 (推荐)
    │
    ▼
ECS应用集群
```

**安全建议**：
- 使用VPN隧道连接邮件服务器
- 或使用IMAP over SSL (993端口)

## 7. 网络安全设计

### 7.1 安全域划分

```
┌─────────────────────────────────────────┐
│            DMZ区 (公网子网)               │
│  • SLB  • NAT  • VPN网关                │
└──────────────┬──────────────────────────┘
               │ [防火墙规则]
┌──────────────▼──────────────────────────┐
│           应用区 (私网子网1)              │
│  • ECS集群  • OpenClaw服务              │
└──────────────┬──────────────────────────┘
               │ [防火墙规则]
┌──────────────▼──────────────────────────┐
│           数据区 (私网子网2)              │
│  • RDS  • Redis  • OSS内网 endpoint     │
└─────────────────────────────────────────┘
```

### 7.2 网络ACL规则

**公网子网ACL**：
```yaml
入站:
  - 允许 0.0.0.0/0 TCP 443 (HTTPS)
  - 允许 0.0.0.0/0 TCP 80 (HTTP)
  - 拒绝其他

出站:
  - 允许全部
```

**私网子网ACL**：
```yaml
入站:
  - 允许 VPC网段 全部端口
  - 允许 192.168.1.0/24 (VPN)
  - 拒绝其他

出站:
  - 允许 VPC网段
  - 允许 Internet (通过NAT)
  - 允许 192.168.1.0/24 (VPN)
```

### 7.3 DDoS防护

**阿里云DDoS防护**：
- 启用基础DDoS防护（免费）
- 可选：DDoS防护包（高级防护）
- 针对SLB和EIP启用防护

### 7.4 WAF防护（可选）

**Web应用防火墙**：
- 防护SLB入口
- 防止SQL注入、XSS等攻击
- 访问控制规则

| 指标 | 监控工具 | 告警阈值 |

## 9. 容灾设计

### 9.1 跨区域容灾（可选）

**生产环境**：华东区域
**容灾环境**：华北区域（可选）

**容灾切换流程**：
1. DNS切换到容灾区域
2. VPN重新连接到容灾VPN网关
3. RDS从库提升为主库
4. 恢复业务

### 9.2 本地高可用

**VPN冗余**：
- 主备VPN网关
- 自动故障切换

**ECS高可用**：
- 多可用区部署
- 健康检查 + 自动恢复
